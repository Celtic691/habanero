Frustrations:

  • Error messaging
  • Having to do the classdef's when I just wanted to create a simple working application
  • Trying to enter decimals in edit form

Error detection:

  • missing log4net.dll (and other important dll's?)
  • missing app.config
  • missing settings in app.config
  • missing classdefs.xml
  • thorough classdefs syntax checking (for subtleties that are still valid xml) - eg. multiplicity, missing subelements
  • missing dtd's
  • class defined in classdefs but no matching class in assembly
  • ?faulty database connection
  • user trying to edit primary key (can automatically make the field read-only)

Error messaging:

  • The error message window requires you to look in the stack trace for the error - ok for developers, but not for users
  • Get "Length cannot be less than zero" when class def components missing
  • Mismatch between classDefs and database brings up datareader error (can happen when you change db but not classdefs)
  • Got error message "ownerClassPropDef" can't be null (while trying to get relationships working). Was a missing ID, or rather a misspelt ID.
